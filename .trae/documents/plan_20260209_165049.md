# 需求自动分析与导入 PingCraft 工具开发计划 (最终版)

## 1. 项目初始化与架构设计

* **项目结构**: Monorepo 结构 (`frontend`, `backend`)。

* **技术栈**:

  * **前端**: Vue 3 + TypeScript + Vite + Element Plus (浅色科技风，无 Emoji/渐变，小圆角)。

  * **后端**: Node.js + Express + LangChain.js。

  * **数据库**: **嵌入式 SeekDB (混合数据库)**。

    * 统一使用 SeekDB 存储向量数据（项目/需求）和关系型数据（用户 Token）。

  * **环境配置**: `.env` 管理 LLM 参数与 PingCode 凭证。

## 2. 前端开发 (Vue 3)

* **UI 规范**: 严格执行浅色科技风、无 Hover 特效、圆角 < 6px。

* **核心功能**:

  * **用户授权**: 登录页引导 PingCode 授权。

  * **数据同步**: “同步数据”按钮，触发从 PingCode 拉取最新数据并更新至本地 SeekDB。

  * **文档上传**: 支持 Markdown/TXT/Word，实时展示解析与分析进度。

  * **需求分析看板**: 展示 LLM 提取的需求详情。

  * **智能匹配**:

    * 基于 SeekDB 检索相似项目。

    * 基于 SeekDB 检索相似历史工作项（查重）。

  * **导入管理**: 确认列表后批量提交至 PingCode。

## 3. 后端开发 (Node.js + Express)

* **数据库设计 (SeekDB)**:

  * `User_Auth`: 存储用户认证信息 (`user_id`, `access_token`, `expires_at`)。

  * `Projects_Vector`: 存储项目信息及向量索引。

  * `WorkItems_Vector`: 存储工作项信息及向量索引。

* **API 接口**:

  * `GET /auth/login`: 启动授权。

  * `GET /auth/callback`: 换取 Token 并存入 SeekDB 的 `User_Auth` 表/集合。

  * `POST /sync-data`: 使用当前 Token 拉取 PingCode 数据 -> 更新 SeekDB 中的项目与工作项向量。

  * `POST /analyze`: 文档 -> 文本提取 -> LangChain Agent -> 结构化数据。

  * `POST /match-project`: 在 SeekDB 中进行语义检索。

  * `POST /import`: 校验 Token 有效性（过期则返回 401）-> 批量创建工作项。

## 4. 核心逻辑实现

* **Token 管理**:

  * 登录成功后，将用户信息存入 SeekDB。

  * 每次请求先查 SeekDB 校验 Token 时效，不进行自动刷新，过期强制重登。

* **LangChain Agent**:

  * 集成 LLM 进行非结构化文本分析。

  * 结合 SeekDB 进行 Retrieval Augmented Generation (RAG) 或单纯的语义搜索辅助。

* **PingCode 集成**:

  * 封装 REST API 调用。

  * 实现“同步”逻辑：Fetch All Projects/Items -> Embed -> Upsert to SeekDB。

## 5. 测试与部署

* **测试**:

  * 单元测试覆盖 SeekDB 的 CRUD 操作（混合数据读写）。

  * E2E 测试验证“授权 -> 同步 -> 分析 -> 导入”全流程。

* **部署**:

  * 编写 `Dockerfile`，确保 SeekDB 嵌入式环境配置正确。

  * 输出《技术文档》与《用户手册》。

## 6. 实施步骤

1. **初始化**: 创建项目，安装 `seekdb`, `express`, `vue` 等依赖。
2. **数据库层**: 初始化 SeekDB，设计用户表与向量集合结构。
3. **授权模块**: 实现 PingCode OAuth，将 Token 存入 SeekDB。
4. **同步模块**: 开发数据拉取与向量化入库功能。
5. **业务模块**: 开发文档解析、LLM 分析、前端交互。
6. **导入模块**: 完成数据回写 PingCode 功能。
7. **交付**: 测试、UI 调整、文档编写。
