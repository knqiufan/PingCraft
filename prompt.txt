我需要开发一个需求自动分析与导入PingCode的Agent工具，具体技术实现要求如下：

**功能需求：**

1. **用户授权登录模块**
   - 实现PingCode OAuth授权流程，用户首次访问页面时自动跳转至PingCode授权页面
   - 授权成功后自动重定向回原页面，并保存用户认证信息
   - 实现Token刷新机制，确保会话持续有效

2. **文档上传与解析模块**
   - 前端实现支持markdown、txt、word格式文件的上传组件
   - 后端接收文档后进行格式解析，提取纯文本内容
   - 集成LangChain v1.x框架调用LLM进行智能分析
   - 定义严格的输出格式规范，包含：项目名称、需求列表、每项需求的预估工时、优先级、描述等字段

3. **智能项目匹配模块**
   - 调用PingCode接口获取用户可访问的项目列表
   - 实现基于文本相似度的算法，自动匹配最相似的项目
   - 提供项目选择器，支持用户手动修改或清空自动匹配结果

4. **工作项管理模块**
   - 实现解析后工作项的可视化展示，支持添加、编辑、删除操作
   - 若项目ID为空，弹出确认对话框提示用户创建新项目
   - 获取项目现有工作项，与新解析的工作项进行智能比对
   - 对相似工作项提供更新/创建/忽略三种操作选项

5. **批量导入模块**
   - 实现批量创建工作项的接口调用
   - 提供导入进度显示和错误处理机制
   - 导入完成后显示成功/失败统计信息

**技术实现要求：**

**前端技术栈：**
- Vue 3 + TypeScript + Vite + pnpm
- Element UI组件库，仅使用其内置icon
- 浅色科技风设计，禁用emoji、禁用篮子渐变色、禁用大于6px圆角、禁用hover效果
- 实现响应式布局，适配不同屏幕尺寸

**后端技术栈：**
- Node.js + Express框架
- 嵌入式SeekDB数据库，参考文档 `seekdb` 
- LangChain v1.x进行Agent编排和LLM调用
- 环境配置通过.env文件管理LLM参数

**接口集成：**
- 参考PingCode接口文档 `pingcode` 实现所有API调用
- 实现接口调用的错误重试机制
- 添加接口调用日志记录功能

**测试与部署：**
- 编写单元测试覆盖核心功能模块
- 实现端到端测试验证完整流程
- 提供Docker容器化部署方案
- 编写详细的技术文档和用户手册

**性能要求：**
- 文档解析响应时间不超过5秒
- 支持最大10MB文档文件上传
- 批量导入工作项时支持每批次50条数据
- 系统并发用户数支持100人以上
